# SEO Fix — ControlApps Brand Name + Canonical URL Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Corregir el brand name "ControlApp" → "ControlApps" en todo el sitio, fijar la URL canónica a `https://apps.controldoc.app` y dejar el proyecto listo para Google Search Console.

**Architecture:** Cambios de contenido/config en archivos existentes sin tocar routing, componentes ni lógica. El env var `NEXT_PUBLIC_APP_URL` en Vercel controla la base URL de todo el SEO (sitemap, canonicals, JSON-LD). El meta tag de verificación GSC se agrega en `app/layout.tsx` cuando el usuario provea el token.

**Tech Stack:** Next.js 14 App Router, TypeScript, Vercel (REST API), Cloudflare DNS

## Task 6: Configurar NEXT_PUBLIC_APP_URL en Vercel

**Files:** ninguno (API call a Vercel)

Este task configura el env var que hace que el sitemap, canonical tags y JSON-LD usen `https://apps.controldoc.app` en lugar del fallback incorrecto.

- [ ] **Step 1: Leer el token de Vercel desde settings**

```bash
node -e "
const s = require(require('os').homedir() + '/.claude/settings.json');
console.log(s.env.VERCEL_TOKEN);
"
```

- [ ] **Step 2: Crear el env var en Vercel (production + preview)**

Reemplazar `VERCEL_TOKEN` con el valor del step anterior:

```bash
curl -s -X POST "https://api.vercel.com/v9/projects/controlapp/env" \
  -H "Authorization: Bearer $(node -e "const s=require(require('os').homedir()+'/.claude/settings.json');process.stdout.write(s.env.VERCEL_TOKEN)")" \
  -H "Content-Type: application/json" \
  -d '{
    "key": "NEXT_PUBLIC_APP_URL",
    "value": "https://apps.controldoc.app",
    "type": "plain",
    "target": ["production", "preview"]
  }' | node -e "
const d=[];process.stdin.on('data',c=>d.push(c));process.stdin.on('end',()=>{
  const r=JSON.parse(Buffer.concat(d).toString());
  console.log(r.error ? 'ERROR: '+r.error.message : 'OK - ID: '+r.id);
})"
```

Esperado: `OK - ID: env_xxxxxxxx`

- [ ] **Step 3: Verificar que el env var quedó registrado**

```bash
curl -s "https://api.vercel.com/v9/projects/controlapp/env" \
  -H "Authorization: Bearer $(node -e "const s=require(require('os').homedir()+'/.claude/settings.json');process.stdout.write(s.env.VERCEL_TOKEN)")" | \
  node -e "
const d=[];process.stdin.on('data',c=>d.push(c));process.stdin.on('end',()=>{
  const r=JSON.parse(Buffer.concat(d).toString());
  const env = (r.envs||[]).find(e=>e.key==='NEXT_PUBLIC_APP_URL');
  console.log(env ? 'Encontrado: '+env.key+' = '+env.value : 'NO encontrado');
})"
```

Esperado: `Encontrado: NEXT_PUBLIC_APP_URL = https://apps.controldoc.app`

## Task 8: Google Search Console (pasos manuales del usuario + código)

**Files:**
- Modify: `app/layout.tsx` (agregar el meta tag de verificación)

Este task requiere que el usuario complete los pasos en el browser y provea el token de verificación.

- [ ] **Step 1: El usuario completa estos pasos en el browser**

```
1. Ir a https://search.google.com/search-console/welcome
2. Elegir "Prefijo de URL"
3. Ingresar: https://apps.controldoc.app
4. Clic en "Continuar"
5. En "Métodos de verificación" elegir "Etiqueta HTML"
6. Copiar el contenido del atributo "content" del meta tag
   (algo como: "abc123XYZ_XXXXXXXXXXXXXXXXXXXXXXXXXXXXX")
7. Proveer ese valor para continuar con este task
```

- [ ] **Step 2: Agregar el meta tag de verificación en app/layout.tsx**

En el objeto `metadata` de `app/layout.tsx`, agregar la clave `verification` después de `icons`:

```typescript
// Después del bloque icons: { ... },
verification: {
  google: "PEGAR_TOKEN_AQUI",
},
```

El bloque `metadata` completo con la adición queda así (sección final):

```typescript
  icons: {
    icon: [
      { url: "/favicon.ico", sizes: "any" },
      { url: "/icon-192.jpg", type: "image/jpeg", sizes: "192x192" },
      { url: "/icon-512.jpg", type: "image/jpeg", sizes: "512x512" },
    ],
    apple: [{ url: "/icon-192.jpg", sizes: "192x192" }],
  },
  appleWebApp: {
    capable: true,
    statusBarStyle: "default",
    title: "ControlApps",
  },
  verification: {
    google: "PEGAR_TOKEN_AQUI",
  },
}
```

- [ ] **Step 3: Verificar que el meta tag aparece en el HTML**

```bash
pnpm build && curl -s "http://localhost:3000" | grep "google-site-verification"
```

Alternativamente, después del deploy:

```bash
curl -s "https://apps.controldoc.app" | grep "google-site-verification"
```

Esperado: `<meta name="google-site-verification" content="PEGAR_TOKEN_AQUI"/>`
